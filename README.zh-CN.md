# Universal Complex Task Automation Framework

> 一个**带人机确认环的复杂多步任务管理框架**，在单个 Claude Code 会话中运行。

[![Version](https://img.shields.io/badge/version-v3.3-blue)]()
[![Architecture](https://img.shields.io/badge/architecture-Factory--inspired-green)]()

## 概述

该框架实现了**多角色模拟**的工厂流水线模式 —— 编排者(Orchestrator)、执行者(Worker)、验证者(Validator) —— 全部运行在**单个 Claude Code 实例**内。它将模糊或不明确的原始需求，转化为系统化执行、可验证的交付物。

适用于满足 **2 条及以上**以下条件的任务：

| 维度 | 含义 |
|-------|--------|
| 📁 文件范围 | ≥3 个文件或跨模块改动 |
| 🧠 知识领域 | 多领域交叉（代码 + 架构 + 测试 + 文档）|
| 🔀 路径确定性 | 多种实现路径，需探索 |
| 🔗 隐藏依赖 | 改 A 会对 B/C/D 产生级联影响 |
| 🔄 完成轮次 | 需要多轮交互才能完成 |
| ✅ 验证维度 | 多维度验证（功能 + 性能 + 安全）|

### 核心公式

```
T = (U ⊙ S ⊙ C) × O × [ (W → Cm → H → V → F)^k ]^m × A × M
```

| 符号 | 含义 |
|--------|---------|
| **T** | 任务输出 — 最终交付物 |
| **U** | 用户意图 — 自然语言输入 |
| **S** | 结构化规格 — 人类可读的规格说明 |
| **C** | 验证合约 — 机器可执行的断言集合 |
| **⊙** | 冻结算子 — 一旦确认，实现阶段不可变更 |
| **O** | 编排者 — 规划、拆解、调度；不碰实现细节 |
| **W** | 执行者 — 单功能实现者 |
| **Cm** | 提交/快照 — 每个功能完成后不可变回滚点 |
| **H** | 交接文档 — 结构化 Markdown 交接物 |
| **V** | 验证者 — 独立检查，每次新上下文 |
| **F** | 修复任务 — 验证失败时由 O 生成 |
| **k** | 收敛迭代 — 单个里程碑内的修复轮数 |
| **m** | 里程碑数 — 阶段门控，各自独立可交付 |
| **A** | 审计日志 — 全链路可追溯 |
| **M** | 记忆 — 跨任务经验累积 |

## 架构

```
                  ┌─────────────────────────────┐
                  │       编排者 (O)             │
                  │  规划 · 拆解 · 调度          │
                  └──────────┬──────────────────┘
                             │
           ┌─────────────────┼─────────────────┐
           │                                   │
    ┌──────▼──────┐                    ┌───────▼───────┐
    │ 执行者 (W)  │                    │ 验证者 (V)    │
    │  实现功能   │                    │  检查结果     │
    └──────┬──────┘                    └───────┬───────┘
           │                                   │
           └─────────────┬─────────────────────┘
                         │
                  ┌──────▼──────┐
                  │  交接文档(H) │
                  │  结构化文字  │
                  └─────────────┘
```

三个角色是**同一会话内的上下文切换** —— 无独立进程。角色间通信**仅通过结构化文件**（交接文档、共享状态、审计日志）。**禁止角色间直接点对点对话。**

## 流水线阶段

### 阶段 0 — 预检查
按 6 个复杂度维度评估任务。若 < 2 条标记为"复杂"，切换至快速执行模式。

### 阶段 0.5 — 状态检测与恢复
初始化前检查是否存在已有任务状态。若上次会话被中断，向用户提供恢复选项：
- **恢复执行** — 从精确的最后状态继续
- **重新开始** — 丢弃进度，从零开始
- **检查详情** — 查看详细状态后再决定

### 阶段 1 — 目标定义 (U → S → C)
三层规格，每层均有人工确认：

1. **U 解析** — 提取显式+隐式需求，写入 `01-user-intent.md`
2. **S 生成** — 结构化规格（功能/非功能/反需求 + 验证需求）→ **人工确认**
3. **C 生成** — 验证合约（机器可执行断言）→ **人工抽检**
4. **功能规划** — 里程碑分解 + 断言映射 → **人工确认**

### 阶段 2 — 工作区初始化
通过 `init-task.sh` 创建工作区，初始化 `state.json` 和所有参考模板。

### 阶段 3 — 里程碑执行循环
每个里程碑（共 m 轮迭代）：
- **M-i.1** O：里程碑启动 — 读取状态、交接文档、记忆；生成功能规划
- **M-i.2** 功能执行循环（每功能 k 轮）：
  - **W** — 执行者实现一个功能
  - **Cm** — 提交快照
  - **H** — 撰写结构化交接文档
  - **V** — 验证者用新上下文独立检查
  - **F** — 若验证失败，O 生成修复任务；循环直至通过
- **M-i.3** 里程碑门禁 — 记忆整理、状态更新、人工检查点

### 阶段 4 — 审计日志
跨切面、仅追加的流，记录每个动作（谁/何时/做了什么/为什么/结果如何）。

### 阶段 5 — 记忆与最终交付
项目级记忆积累、最终验证扫描、交付物组装。

### 阶段 6 — 优雅终止
防跑飞循环保护 — 全局迭代预算、收敛检查、人工终止信号。

### 阶段 7 — 挂起与恢复
用户发起的挂起操作，精确保存执行上下文供未来恢复。

## 快速开始

### 通过 Claude Code 直接调用

```
/universal-complex-task "将认证模块重构为支持 OAuth2，
拆分为 3 个里程碑，使用 TDD 并包含集成测试"
```

技能会自动填充所有文件；你只需在阶段 1 的 **3 个人工检查点** 确认即可。

### 使用 init-task.sh 脚本

```bash
# 将此仓库放入 Claude Code 的 skills 目录
# 然后初始化一个任务工作区：
./scripts/init-task.sh 我的任务名称

# 指定项目根目录：
PROJECT_ROOT=/path/to/project ./init-task.sh 我的任务名称

# 自动确认检测到的根目录：
CONFIRM_ROOT=1 ./init-task.sh 我的任务名称
```

> 脚本创建的是空工作区供手动填写需求。如需全自动化，通过 Claude Code 直接调用该技能。

│       ├── 02-structured-spec.md    # S — 结构化规格
│       ├── 03-validation-contract.md # C — 机器可执行合约

│       ├── 05-audit-log.md          # A — 仅追加审计跟踪
│       ├── 06-memory.md             # M — 任务级记忆

## 设计原则

| 原则 | 说明 |
|-----------|-------------|
| **🧊 冻结算子** | U→S→C 一旦人工确认，实现阶段不可变更规格 |
| **🔄 串行写入** | 同一时间仅一个写者 — 无合并冲突 |
| **👁️ 新上下文** | 验证者仅读取合约+已提交产物；不受实现过程偏见影响 |
| **📋 结构化交接** | 角色间不传递会话记忆 — 仅通过结构化 Markdown |
| **📎 仅追加审计** | 全链路可追溯 — 每个动作均有记录，绝不删除 |
| **🧠 预载记忆** | 跨任务经验在角色行动前加载，而非事后才记录 |
| **🛑 禁止自我修复** | 执行者绝不自行修复 BUG — O 根据验证者发现生成修复任务 |

## 与 Factory 架构的关系

本框架**受 Factory 多智能体架构启发**，但适用于**单实例**环境：

| 维度 | Factory | 本框架 |
|--------|---------|-------------|
| 进程 | 多模型/多进程 | 单个 Claude Code 实例 |
| 角色 | 独立智能体 | 单会话内上下文切换 |
| 通信 | 智能体间对话 | 仅结构化文件 |
| 并发 | 并行 | 串行写入，并行读取 |
