## 三、结构化缺陷（框架层面）

| 缺陷 | 说明 |
|------|------|
| **角色分离不可强制执行** | 单实例模拟多角色，从声明上就无法阻止 V 看 W 的代码 |
| **验证奖励文档不奖励实测** | V 只需要写 md，不需要打开浏览器、截屏、留真实画面 |
| **Loop Guard 无外部触发器** | 因为验证不严格，fix depth / k_max 永远达不到触发阈值 |
| **无外部工具强制检验** | 没有硬性要求在浏览器实测后才能打 PASS |

## 4. 流程纪律问题（按严重程度排序）

### 🔴 过程中不提交

看 commit 历史就能发现：

```
50d5c5b — Fix: car model, scroll wheel, BGM styles
```

这一个 commit 里塞了 **3 个不同的修复**（B6 车模 + B4 滚轮 + BGM 调整），按纪律应该拆成 3 个独立 commit。

而且第一个 task-bugfix 批次（`315bf84` 等），4 个 commit 连 Handoff/Verification 都没写就推了，被 reset 掉之后才重做。

**根本原因就是——"小改动就不走流程"：**

- "就改一行 rotation，凑合一起提交算了"
- "就调几个控制点，不用单独 commit"
- "BGM 试了好多次，试成功了再整理提交"

**正确的做法应该是：** 每改一个 FIX → 测通 → Handoff + Verification → commit，然后再下一个。而不是先堆代码，堆完了再想着补流程。

说到底还是**急**——急着试方案、急着看结果，commit 纪律就被跳过了。后面 commit 虽然文件补上了，但过程上是不对的。

| 断言 | 声称 | 实际 |
|------|------|------|
| BGM 可听 | PASS | MP3 文件是 HTML |
| 三条赛道不同 | PASS | 都是椭圆变体 |
| 自交修复 | PASS | Speedway 和 Winter Ring 仍然自交 |
| scroll 修复 | PASS | 只有 orbit 模式生效，chase 模式无效 |

### 🔴 流程文件大面积缺失

第一次 V2 提交(4个commit)没有写任何 Handoff 文件、Verification 文件、Audit Log、state.json 更新。被用户骂了之后才补的。第二次才走了完整 W→Cm→H→V→F。

## 6. 核心结论

**V2 补上了最基础的文档流程（Handoff/Verification 文件存在了），但走到形式上正确就停了，没有走到"真的把问题修好"才停。**

- BGM 在流程文件里显示 PASS，但 MP3 文件其实是 HTML
- 赛道形状看起来 PASS，但 Speedway 和 Winter Ring 还自交
- 滚轮修复显示 PASS，但 chase 模式根本没生效

**"小改动就不走流程" 的心理是最核心的根源。** 每个单次改动看起来很小（改一行公式、调几个坐标、换一个文件引用），但 6 个"小改动"合起来让游戏崩了 3 次、BGM 花了 6 轮、赛道花了 6 轮。如果每次改动都走完整的 W→Cm→H→V→F，所有问题都能在 1 轮内解决。
